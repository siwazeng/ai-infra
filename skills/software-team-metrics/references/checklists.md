# 软件团队度量清单与模板

## 1. 启动前检查清单

在设计指标前，先确认这些问题：

1. 我们真正想改善什么？
2. 这些问题更偏个人、团队、项目，还是发布结果？
3. 这次度量是为了理解、改进，还是已经被误用成考核？
4. 数据源在哪里？口径是否一致？
5. 谁来维护？谁来解释？谁来决策？
6. 这些指标看完之后，会触发什么动作？
7. 团队是否知道这件事的目的？
8. 是否存在明显不可比的上下文差异？

## 2. 指标设计卡

为每个候选指标，至少写清以下信息：

- 指标名：`____`
- 目标：`它帮助我们理解什么`
- 类型：技能 / 响应 / 价值
- 观察对象：个人 / 团队 / 项目 / 发布
- 时间窗口：`____`
- 数据来源：`____`
- 口径说明：`____`
- 负责人：`____`
- 看完之后的动作：`____`
- 风险：`这个指标最容易被误解成什么`

## 3. 首批试点清单

### 范围

- 是否控制在 3～7 个指标内？
- 是否只选一个团队或一个项目先试？
- 是否给出了固定试点周期？

### 节奏

- 是否明确周更新 / 双周更新 / 月复盘？
- 是否约定了谁整理记分卡？
- 是否安排了固定讨论时间？

### 成功标准

- 是否定义了“这个试点值得继续”的判断标准？
- 是否约定了停用或改口径的触发条件？

## 4. 指标组合模板

### 场景 A：交付与质量

- 技能：得分 / 助攻 / 错误 / 加减
- 响应：罚球 / 输场
- 价值：效率 / 守场

### 场景 B：增长与用户响应

- 响应：胜场 / 胜率 / 输场 / 增益
- 技能：抢断 / 火力
- 价值：胜场份额 / 领先份额

### 场景 C：团队协作与结构

- 技能：助攻 / 活动范围 / 救援
- 价值：团队合作 / 影响力 / 战斗力

### 场景 D：招聘与团队补位

- 技能：得分 / 火力 / 抢断 / 活动范围 / 错误
- 价值：守场 / 爆发力 / 团队合作

## 5. 复盘问题清单

在看记分卡时，优先回答这些问题：

1. 哪些指标变化最大？
2. 这是偶发波动，还是持续趋势？
3. 指标之间是否互相印证？
4. 有没有明显的峰值、谷值和异常点？
5. 这些变化和团队最近发生的什么事有关？
6. 这里最值得追问的一个问题是什么？
7. 这轮复盘之后要改变什么？

## 6. 一对一讨论模板

如果要和个体讨论度量，建议按这个结构：

1. 先确认角色和上下文。
2. 再看趋势，不先看排名。
3. 先讲模式，再讲判断。
4. 先问本人解释，再给建议。
5. 最后确定一个可执行改进动作。

可直接使用的话术模板：

- `我不是想用这个数字给你定性，我想先理解这里发生了什么。`
- `这段时间你的____变化很明显，你觉得最主要的原因是什么？`
- `如果下个周期只改善一件事，你认为最值得改的是哪一项？`

## 7. 团队角色诊断清单

判断团队结构时，先看这些问题：

1. 团队里谁在组织进攻？
2. 谁是主要得分手？
3. 谁在做防守和救火？
4. 谁承担跨领域工作？
5. 谁在稳定地支持和激励他人？
6. 哪类角色明显缺位？
7. 哪类角色过度集中在个别人身上？

## 8. 招聘 benchmark 模板

### 目标角色

- 角色名：`____`
- 当前团队缺口：`____`
- 目标作用：`____`

### 关键维度

- 得分 / 火力 / 活动范围 / 错误 / 抢断 / 团队合作 / 守场：`选 3~5 个`

### 面试问题模板

- 如果看主动性：
  - `你主动发现并解决过什么没有人明确要求你处理的问题？`
- 如果看协作：
  - `你最近一次显著帮助队友推进工作是什么？`
- 如果看防守：
  - `你处理过最关键的一次线上问题是什么？你如何判断优先级？`
- 如果看多能：
  - `你最近一年跨过哪些不同领域或职责？`

## 10. 反滥用检查清单

上线任何度量机制前，逐条确认：

1. 这个系统是否被误当成个人排名器？
2. 是否存在单指标决定人事的风险？
3. 团队是否知道指标的口径和目的？
4. 指标是否允许被质疑、被修正？
5. 是否给出了“不用于什么”的明确边界？
6. 是否把角色差异、阶段差异写进了解读规则？

## 11. 首月动作模板

如果要从零开始在一个软件团队里引入度量，首月默认做这些事：

1. 写清唯一目标和边界。
2. 找 sponsor 和 1~3 人焦点组。
3. 选 3~7 个首批指标。
4. 定口径、数据源、观察窗口和负责人。
5. 做出第一版记分卡。
6. 连续跑 2~4 个周期，不急着下结论。
7. 做一次试点复盘，决定保留、修改或停用哪些指标。
